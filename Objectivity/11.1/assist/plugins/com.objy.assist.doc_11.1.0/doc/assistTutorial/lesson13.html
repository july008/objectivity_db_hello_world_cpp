Lesson 13: Finding Persistent Objects
Storing persistent objects in a federated database is of no use unless they can be found again. Objectivity/DB supports a variety of ways to find persistent objects. A successful search operation obtains the OID(s) of one or more persistent objects. Depending on the Objectivity programming interface, such operations may also lock the found objects and obtain memory representations of them.
In some cases, you must prepare persistent objects to be found—for example, by naming the objects or by organizing them in various kinds of collections. An application can then look up such objects by name or by an identifying key within the collection.
In other cases, the search operations exploit a federated database’s object model. For example when persistent objects are linked by reference attributes or relationships, an application can follow the links from one object to referenced or related objects.
You can also perform predicate queries to find objects based on their content—that is, by the values of their attributes.
For complete information about finding objects from an application, see the documentation for your Objectivity programming interface.
Querying objects
Sometimes you need to find persistent objects based on the content of their data. For example, you might want to find all available category 3 or category 4 cars in the fleet of a particular region.
A predicate query compares a group of persistent objects against a condition and selects only those objects that satisfy the condition:
■
The tested objects must all be of the same class—for example, all Car objects.
■
You specify the condition to be tested as a predicate string—a string composed of expressions in the Objectivity/DB predicate-query language (PQL).
Assist provides a Query editor that helps you build predicate strings for filtering the persistent objects of a class in the federated database.
From the Query editor, you can access a simple query builder that helps you quickly qualify objects using a subset of the PQL functionality. For example, you can qualify objects based on whether attribute values are equal-to or less-than/greater-than the values you provide.
For more advanced queries that use the full range of PQL functionality, you can access the advanced predicate builder.

The following steps use Assist to find all cars in all of the regional companies. This means querying for all objects of class Car in the federated database.
Objectivity/DB uses the federated database’s placement model to restrict the search to just the containers in which Cars reside, instead of searching every container in the federated database.

Querying a string attribute
Now let’s find all the available cars in the Australian regional company. To do this, we will create a predicate that compares the string attribute rentalCompany.region using regular expression matching.
Assist provides an advanced predicate builder that supports all the PQL capabilities, including regular expression matching.
1.    
Click the Advanced button to start building the predicate.
2.    
Double click the available attribute to add it to the Predicate field.
Notice the Class Context button in case you want to access attributes of a different class for building your predicate; this won’t be necessary for the tutorial, though.
3.    
Click the Built-in Operators tab, scroll down, and double click the relational equality (==) operator to add it to the Predicate field.
The top of the advanced query builder shows information about the state of the predicate, which is currently incomplete.
Notice that the cursor is inside the empty parentheses in the Predicate field. The predicate builder uses the location of the cursor as the insertion point for adding the next PQL elements. It is a good idea to check the cursor location before adding to your query.
4.    
Click the Literals tab, double click on Boolean, select the value true, and click Finish.

Click to place the cursor after the right most closing parenthesis in the Predicate field.
6.    
Click the Built-in Operators tab and double click on the logical conjunction (&&) operator.
The predicate now looks like this:
7.    
Double click on the regular expression match, case-insensitive (=~~) operator in the Built-in Operators tab.
The predicate now looks like this:
8.    
Click the Attributes tab, expand the rentalCompany attribute, and double click the rentalCompany.region string attribute.
The predicate now looks like this:
9.    
Place the cursor between the empty parentheses on the right, click the Literals tab, and double click the regular expression literal (RegEx).
Regular expressions allow you to insert special characters in a comparison string. In this case, we want to match every region that starts with the characters Aus, no matter what the rest of the string contains. The special characters .* (period followed by asterisk) match any number of arbitrary characters.
10.    
In the Regular Expression Literal Value field, enter the following characters. You can use any combination of lowercase or uppercase characters:
Aus.*
Do not enclose this string in double-quotes—Assist will do that for you.
11.    
Click Finish to close the Regular Expression Literal dialog, then click Finish in the advanced predicate builder.
12.    
Click the Perform Query button in the Query editor.
You can verify that the found cars belong to the Australian regional company by showing the region.

Saving a query
After creating a useful predicate query, you can save it for future use.
1.    
In the Query editor, click Save Predicate.
2.    
In the dialog that appears, enter a name for the query and click Finish.
The named predicate is saved and is associated with the Assist workspace you specified when you started Assist.
You can display information about the current Assist workspace by hovering over the Current Project label.
 
Loading a saved query
1.    
In the Query editor, click the browse button next to Predicate.
2.    
In the dialog that appears, double click the name of a previously saved predicate.
3.    
Click Perform Query in the Query editor.
