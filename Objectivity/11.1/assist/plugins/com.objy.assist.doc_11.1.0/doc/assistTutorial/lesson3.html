Lesson 3: What is a Federated Database?
With a traditional DBMS, you store related data in a single database. With Objectivity/DB, you typically distribute related data in many individual databases, which are members of a single federated database (also called a federation). You can look at a list of the actual database files that belong to the RentalCompany federated database. The database files have the suffix .DB.

On Windows, use your Windows Explorer to view the files in the installDir\samples\objydbTutorial folder.

ls
A federated database makes distribution of data across a network possible by maintaining a catalog of the individual databases that belong to the federation. The catalog maps logical database information (names and identifiers) to physical file locations. The catalog resides in a special system database.
So your applications need not manage hundreds or even thousands of database file names—instead, each database is accessed through its logical name, and Objectivity/DB uses the catalog to find the physical location of the database’s file. This distinction between logical and physical names enables database files to be moved to other storage locations without recompiling the application.
The federated database provides unified management of database files and a collection of other Objectivity/DB resources (files and processes):
■
A boot file gives applications the information they need to connect to the federated database. The boot file for the tutorial example is RentalCompany.boot.
■
A system database maintains internal Objectivity/DB information such as the global catalog of databases. The system-database file for the tutorial example is RentalCompany.fdb.
■
A lock server is a process that manages concurrent access to a federated database’s data.
■
Data servers enable applications to obtain data from distributed databases on local or remote hosts.
■
A journal directory (by default, the directory containing the federated database) contains files that maintain information for recovering transactions.
You’ll learn more about each of these resources in later lessons.
