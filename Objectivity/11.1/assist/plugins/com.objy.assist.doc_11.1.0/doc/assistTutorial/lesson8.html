Lesson 8: Browsing Data
In the FD Project Explorer:
►
Make sure Federated Database (RentalCompany) is expanded. Then click to expand +Data.
A federated database’s data is stored in one or more databases, so expanding Data shows a list of databases. Each entry starts with Database: and shows a database’s OID and system name.
The Database: entries under Data look similar to the DB: entries under Catalog > Databases. However, the two kinds of entries differ as follows:
■
Each DB: entry leads you to catalog information. This includes file-related information about the physical storage of the database.
■
Each Database: entry under Data leads you to a logical view of your persistent objects, independent of their physical location. Each Database: entry is the starting point for browsing to persistent objects.
In the FD Project Explorer:
►
Click to expand the +Database: entry for the pmV2-Region_G1_1 database.
A database’s data is stored in one or more containers, so expanding a Database: entry shows a list of containers. Containers are a special kind of object that subdivide a database. Every database has at least one container, and typically has many more. In this example, the pmV2-Region_G1_1 database has containers for a particular regional company and the various entities within it.
Each container has an identifier and a system name. We’ll learn more about containers later.
In the FD Project Explorer:
►
Click to expand the +Container: entry for the container called pmV2-Fleet_P1_1.
Expanding a Container: entry in the FD Project Explorer lists the persistent objects in that container. Persistent objects are the fundamental units of data storage. Each persistent object has an object identifier and is an instance of a class in the schema. (You may need to scroll to the right or widen the FD Project Explorer view to see the object identifiers.) In this example, the pmV2-Fleet_P1_1 container contains one Car or Truck object for each car or truck in the regional company’s fleet of vehicles.
A container can store millions of persistent objects. By default, Assist displays 20 objects at a time in order of their object identifiers, and adds more on request. (A later lesson will show you how to form a query to specify other subsets of objects.)
In the FD Project Explorer:
1.    
Scroll down (if necessary) to see the last Object: entry currently displayed in the list.
2.    
Double-click the down-arrow at the end of the list to show more objects.
Alternatively, you can right-click the down-arrow and select Next 200 from the pop-up menu.
More on browsing
So far, we’ve browsed the federated database’s data by expanding entries in the FD Project Explorer. As an alternative, you can inspect information in the editor area of the Assist window.
In the FD Project Explorer:
1.    
Scroll back up to Data and double-click on it.
The editor area displays an overview of the federated database’s properties and shows a list of databases.
2.    
In the editor area, double-click on the entry for the pmV2-Region_G3_1 database in the Databases box.
The editor area displays an overview of this database’s properties and shows a list of containers.
3.    
In the Containers box in the editor area, double-click on the entry for the container whose system name is pmV2-Fleet_P3_1.
The editor area displays an overview of this container’s properties and shows a list of persistent objects.
This container contains Car and Truck objects representing the vehicles in the rental fleet of a second regional company.
By default, the first 20 persistent objects are displayed, sorted by their object identifiers.
4.    
Double-click the down-arrow at the end of the list to show more objects.
Alternatively, right-click the down-arrow and select an item from the pop-up menu.
You can switch back and forth between editors by single-clicking the tabs on the top edge of the editor area.
To close an editor, move the mouse pointer over the editor’s tab and click the X symbol.
