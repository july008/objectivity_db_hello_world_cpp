Lesson 5: Opening a Federated Database
Now let’s look at an existing federated database. This federated database was structured and populated by a simple example application that models a vehicle-rental business consisting of three regional companies—one in the United States, one in Europe, and one in Australia.
You create an FD project for each federated database you want to manage through Assist. An FD project handles the details for representing a federated database. (You can also manage a federated database directly using Objectivity/DB’s command-line tools. No FD Project is needed in this case.)
In Assist, create a new FD project for the RentalCompany federated database:
1.    
Click File > Open FD.
2.    
In the FD Project Specification dialog, make sure Existing FD is selected.
3.    
In the Boot File field, browse to the boot file for the federated database.
On Windows:
installDir\samples\objydbTutorial\RentalCompany.boot
On UNIX and Macintosh (for example):
homeDir/objydbTutorial/RentalCompany.boot
The boot file contains information that enables an application (such as Assist) to start its interaction with the federated database.
4.    
Click Next.
5.    
In the Project Name field, enter TutorialProject and click Finish.
The FD Project Explorer now lists a new FD project called TutorialProject.
 
You can also start Assist on the command line and specify a boot file to open with the -bootfile argument. For example, the following command starts Assist and creates an FD project for the RentalCompany federated database on Windows:
ooassist.exe -bootfile RentalCompany.boot
