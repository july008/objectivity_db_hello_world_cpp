Lesson 1: Getting Started
You will compare a native C++ program with an Objectivity/C++ program. You will then view the federated database created in the Objectivity/C++ example using Objectivity/Assist.
Native C++ versus Objectivity/C++
In native C++, you can access objects through pointers. In Objectivity/C++, you access persistent objects through handles (smart pointers).
Native C++:
Object* objP; // pointer to object
objP = new Object;
objP->set_some_value();
Objectivity/C++:
ooHandle(Object) objH; // Handle classes are parameterized
objH = new Object;
objH->set_some_value();
Handles are classes and, as such, have methods. So an instance of the handle class can invoke its methods by use of the dot operator (.), for example:
objH.nameObj(â€¦);
A native C++ example
Perform the following steps to inspect and run the native C++ example, a simple C++ program that creates 1,000 objects. This example uses the Objectivity/DB class ooObj from the oo.h include file as the type of object to create.
1.    
Browse to the installDir\samples\cxx\tutorial\example1.1 folder.
2.    
Inspect main.cpp in example1.1\src.
Notice the use of the pointer oRef, which is declared using the previously defined macro Ref(A). It is used with the new statement and it is used with the dereference operator to call a method on the object. Next, notice the file handling code that opens the file, writes the objects to the file, and closes the file.
3.    
Build the project using the provided Makefile or Visual Studio project files.
4.    
Run the program.
An Objectivity/C++ example
Perform the following steps to inspect and run the Objectivity/C++ example, a simple Objectivity/C++ program that creates 1,000 persistent objects and stores them in a federated database. This example uses the Objectivity/DB class ooObj from the oo.h include file as the type of object to create.
1.    
Navigate to the installDir\samples\cxx\tutorial\example1.2 folder.
2.    
Inspect main.cpp in example1.2\src.
The application uses Objectivity handles in the definition of the macro under Ref(A), and the overloaded new operator to create an instance of ooObj. Furthermore, the file handling code has been replaced with some Objectivity/C++ specific code. The Session constructor takes care of Objectivity startup in the statement static Session session("data/example1.2.boot"); the destructor takes care of Objectivity shutdown. Notice there is no specific code to write the objects to the database.
3.    
Build the project using the provided Makefile or Visual Studio project files.
This will create the federated database and compile the database application.
4.    
Browse the contents of example1.2\data.
You will see the boot file example1.2.boot and the federated database file example1.2.fdb.
5.    
Run the program.
6.    
Browse the contents of example1.2\data.
You will see two new files have been created. pmPM_V1-INTERNAL_G1_1.example1.2.DB contains internal files used by Objectivity/DB. pmV1-Default_1.example1.2.DB is the first database within the federated database.
Viewing the federated database
1.    
Start Objectivity/Assist.
Note: If you have not already done so, complete Setting Up Objectivity/Assist For C++ Development in Setting Up before proceeding.
2.    
In Objectivty/Assist, click on File > Open FD to create a new FD Project for the federated database created in the Objectivity/C++ example.
3.    
In the FD Project Specification dialog, make sure Existing FD is selected.
4.    
In the Boot File field, browse to the boot file for the federated database:
installDir\samples\cxx\tutorial\example1.2\data\example1.2.boot
5.    
Click Next.
6.    
Enter example1.2 for the Project name field, and click Finish.
The FD Project Explorer now lists a new FD project called example1.2.
7.    
View the contents of the federated database using Objectivity/Assist.
In the FD Project Explorer:
a.
Click to expand example1.2â€”that is, expand the entry by clicking the symbol.
b.
Click to expand Federated Database (example1.2).
c.
Click to expand Catalog.
d.
Click to expand Databases to see the catalog list of databases.
8.    
Double-click on the Databases entry pmV1-Default_1 to view details of the database in the editor area.
9.    
Browse the data of the federated database.
In the FD Project Explorer:
a.
Click to expand Data.
b.
Click to expand the Database entry pmV1-Default_1.
c.
Click to expand the Container entry to get a list of objects contained in the container.
10.    
Double-click on any Object entry to view additional information about the object in the editor area.
