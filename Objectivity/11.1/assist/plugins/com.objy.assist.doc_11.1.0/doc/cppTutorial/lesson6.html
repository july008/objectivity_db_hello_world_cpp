Lesson 6: Defining Associations
Introduction
In this lesson you will learn how to define, create, navigate and delete associations.
Association properties
Associations are used to link pairs of object instances together; a source object is linked to the destination object. Associations are defined in the class definitions in the DDL files. In a particular class definition, the paired class is the related class.
An association can be either bidirectional (two way, where both source and destination objects know of the association) or unidirectional (one way, where only the source object knows of the association; the destination object does not know it is being referenced). Bidirectional associations maintain referential integrity. When an object is deleted, its associations are deleted and each inverse association is removed from the related objects.
Associations also have cardinality, which means they can be to-one or to-many (from the source to the destination). Bidirectional associations can be one-to-one, many-to-many, one-to-many or many-to-one, depending on the cardinalities from both source and destination.
Once objects are associated with each other, these associations can be used to support navigation and navigational queries.
By default, all associations for a particular object use the same storage. This is convenient for both Objectivity/DB and the database developer. However this approach requires more storage space, which can impact application performance, and mixes many different associations together.
Inline associations
Inline associations address the storage problems and their impact on performance. To-one inline associations are embedded as data members of an object. Every to-many inline association is placed in its own array.
There are two types of inline associations, standard and short. A standard inline association uses an 8-byte object identifier to refer to the destination object; a short inline association uses a 4-byte object identifier to refer to the destination object. A short inline association uses less storage space to maintain the association, resulting in better runtime performance. However, you can only use a short inline association if every destination object is in the same container as the source object.
